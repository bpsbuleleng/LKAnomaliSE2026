Kelola Rule

Satu rule = satu anomali: message yang tampil + kondisi when pendeteksinya. Nonaktifkan = rule berhenti dijalankan saat submit, baris tidak dihapus.

Tambah Rule

Boleh diubah kapan saja (huruf/angka/underscore/strip), mis. K_status_cerai — kosongkan saat menambah rule baru untuk id otomatis. Mengubah ID tidak memengaruhi anomali yang sudah tersimpan di record lama.

⚠ Rule kompleks (memakai roster / kondisi bersarang) — diedit langsung sebagai JSON.

Mode JSON — untuk kondisi rumit: AND/OR dicampur berkurung (nested) atau roster.

Kombinator: all/any (boleh bersarang — all/any di dalam all/any = kurung), roster_any/roster_all/roster_count + condition. Operator: ==, !=, >, >=, <, <=, in, not_in, empty, not_empty, regex. Bandingkan antar-field pakai "field2".
Contoh kurung a==1 AND (b>0 OR c<0): {"all":[{"field":"a","op":"==","value":1},{"any":[{"field":"b","op":">","value":0},{"field":"c","op":"<","value":0}]}]}
